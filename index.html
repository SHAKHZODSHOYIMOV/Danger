Naming DANGEROUS ☢ PYTHON ☢

  • Danger Python is a tool used to automate code review tasks during the CI process.
  • The tool is used to create and execute a Dangerfile.py in the root directory of a repository, which sets up rules for the review process. The danger-python script is executed after the test suite in the CI process
  • The tool helps to automate and streamline common code review chores, making it easier for teams to review and approve code changes.

WHAT IS DANGER PYTHON?

  • Danger runs during your CI process, and gives teams the chance to automate common code review chores.
  • This provides another logical step in your build, through this Danger can help lint your rote tasks in daily code review.
  • You can use Danger to codify your teams norms. Leaving humans to think about harder problems.
  • This happens by Danger leaving messages inside your PRs based on rules that you write in Python.
  • Over time, as rules are adhered to, the message is amended to reflect the current state of the code review.
